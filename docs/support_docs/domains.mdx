# Custom Domains

Connect your own domain to Vibey funnels. Automatic SSL, DNS verification, and full branding control.

## Why Use a Custom Domain?

### Branding

Your domain = your brand.

**vibeos.com/yourfunnel** → Looks like Vibey
**yourdomain.com** → Looks like you

### Trust

Custom domains increase conversion rates.

**Stats:**
- 30-40% higher trust perception
- Better click-through rates in emails
- Professional appearance
- Improved SEO

### Ownership

You own the domain. No platform lock-in.

**Benefits:**
- Move to another platform anytime (keep domain)
- Full control over DNS
- Your asset, your rules

## Getting Started

### What You Need

1. **A domain name** - Buy from any registrar (GoDaddy, Namecheap, Cloudflare, etc.)
2. **Access to DNS settings** - You'll add records
3. **5-10 minutes** - For setup

### Domain Types Supported

**Apex Domains:**
- `example.com` ✅
- `yourbusiness.com` ✅

**Subdomains:**
- `www.example.com` ✅
- `funnel.example.com` ✅
- `get.example.com` ✅

**Not Supported:**
- Email addresses (`name@example.com`) ❌
- Subfolders (`example.com/page`) ❌

<Steps>

  </Step>

## DNS Records Explained

You'll add 2-3 DNS records to your domain provider.

### A Record (Apex Domain)

Points your root domain to Vibey servers.

**Example:**
```
Type: A
Name: @ (or leave blank)
Value: 76.76.21.21 (Vibey IP - example)
TTL: 3600 (or Auto)
```

**Use Case:** `example.com` → Vibey funnel

### CNAME Record (Subdomain)

Points a subdomain to Vibey.

**Example:**
```
Type: CNAME
Name: www (or funnel, or get, etc.)
Value: cname.vibeos.com
TTL: 3600 (or Auto)
```

**Use Case:** `www.example.com` → Vibey funnel

### TXT Record (Verification)

Proves you own the domain.

**Example:**
```
Type: TXT
Name: _vibey-verification (or @ for root)
Value: vibey-verify-abc123xyz789
TTL: 3600 (or Auto)
```

**Use Case:** Verification only (can delete after verified)

### Cloudflare DNS Setup

Cloudflare is recommended for speed and reliability.

<Steps>
  <Step title="Log into Cloudflare">
    Go to [dash.cloudflare.com](https://dash.cloudflare.com) and sign in
  </Step>

  <Step title="Select Your Domain">
    From the dashboard, click the domain you want to configure
  </Step>

  <Step title="Navigate to DNS">
    Click **DNS** in the left sidebar, then click **Records** tab
  </Step>

  <Step title="Add A Record (for apex domain)">
    Click **Add record** button

    **Fill in:**
    - Type: `A`
    - Name: `@` (leave blank if field is auto-filled)
    - IPv4 address: Paste IP from Vibey (e.g., `76.76.21.21`)
    - Proxy status: **DNS only** (gray cloud icon)
    - TTL: `Auto` (or 3600)
    - Click **Save**
  </Step>

  <Step title="Add CNAME Record (for www subdomain)">
    Click **Add record** button again

    **Fill in:**
    - Type: `CNAME`
    - Name: `www` (or your subdomain like `funnel`, `get`, etc.)
    - Target: Paste CNAME from Vibey (e.g., `cname.vibeos.com`)
    - Proxy status: **DNS only** (gray cloud icon)
    - TTL: `Auto` (or 3600)
    - Click **Save**
  </Step>

  <Step title="Add TXT Record (for verification)">
    Click **Add record** button again

    **Fill in:**
    - Type: `TXT`
    - Name: `_vibey-verification` (or `@` if Vibey specifies root)
    - Content: Paste verification code from Vibey (e.g., `vibey-verify-abc123xyz789`)
    - TTL: `Auto` (or 3600)
    - Click **Save**
  </Step>

  <Step title="Verify Proxy Status">
    **CRITICAL:** Ensure all records show gray cloud (DNS only), NOT orange cloud (Proxied)

    If orange cloud is showing:
    - Click the orange cloud icon on each record
    - It will toggle to gray (DNS only)
    - This is required for SSL to work properly
  </Step>
</Steps>

**Common Issues:**

<Accordion title="Orange cloud (Proxied) breaking SSL">
  **Problem:** Vibey can't provision SSL when Cloudflare proxy is enabled

  **Solution:**
  1. Find your Vibey DNS records in Cloudflare
  2. Click the orange cloud icon on each record
  3. It should turn gray (DNS only)
  4. Wait 5 minutes, then retry verification in Vibey
</Accordion>

<Accordion title="DNS not propagating">
  **Problem:** Vibey not detecting DNS records after 30+ minutes

  **Solution:**
  1. Check records are saved (refresh Cloudflare page)
  2. Use [whatsmydns.net](https://whatsmydns.net) to check propagation globally
  3. Wait up to 60 minutes for full propagation
  4. Ensure no typos in record values
</Accordion>

<Accordion title="'Record already exists' error">
  **Problem:** Cloudflare shows error when adding record

  **Solution:**
  1. Check if a similar record already exists (scroll down)
  2. Delete or edit the existing record instead of adding new
  3. If managing multiple subdomains, ensure unique names
</Accordion>

### GoDaddy DNS Setup

<Steps>
  <Step title="Log into GoDaddy">
    Go to [godaddy.com](https://godaddy.com) > Sign in > Click your profile icon > **My Products**
  </Step>

  <Step title="Select Domain">
    Find your domain in the list, click the **DNS** button next to it (or click domain name > **Manage DNS**)
  </Step>

  <Step title="View DNS Records">
    Scroll down to the **DNS Records** section. You'll see existing records (A, CNAME, MX, etc.)
  </Step>

  <Step title="Add A Record (for apex domain)">
    Click **Add** button in DNS Records section

    **Fill in:**
    - Type: Select `A` from dropdown
    - Name: Type `@` (GoDaddy auto-adds your domain, just use @)
    - Value: Paste IP from Vibey (e.g., `76.76.21.21`)
    - TTL: Leave as `1 Hour` (or choose `Custom` > 3600 seconds)
    - Click **Save**
  </Step>

  <Step title="Add CNAME Record (for www subdomain)">
    Click **Add** button again

    **Fill in:**
    - Type: Select `CNAME` from dropdown
    - Name: Type `www` (or your subdomain)
    - Value: Paste CNAME from Vibey (e.g., `cname.vibeos.com`)
    - TTL: Leave as `1 Hour`
    - Click **Save**

    **Note:** GoDaddy may show your full domain in the Name field (`www.example.com`) - that's normal, just ensure `www` part is correct
  </Step>

  <Step title="Add TXT Record (for verification)">
    Click **Add** button again

    **Fill in:**
    - Type: Select `TXT` from dropdown
    - Name: Type `_vibey-verification` (or `@` if Vibey specifies root)
    - Value: Paste verification code from Vibey (e.g., `vibey-verify-abc123xyz789`)
    - TTL: Leave as `1 Hour`
    - Click **Save**
  </Step>

  <Step title="Confirm Changes">
    All three records should now appear in your DNS Records list. Changes save automatically.
  </Step>
</Steps>

**Common Issues:**

<Accordion title="GoDaddy auto-adds domain to Name field">
  **Problem:** When you type `www`, GoDaddy shows `www.example.com.example.com`

  **Solution:**
  - Just type `www` or `@` in the Name field
  - GoDaddy automatically appends your domain
  - If you see duplicate domain, edit the record and remove the extra domain
</Accordion>

<Accordion title="Can't add A record for @ (root)">
  **Problem:** GoDaddy shows error "A record for @ already exists"

  **Solution:**
  1. Scroll through existing records to find the A record for `@`
  2. Click the pencil icon to **Edit** that record
  3. Change the Value to Vibey's IP address
  4. Save the edited record (don't add new one)
</Accordion>

<Accordion title="Domain parking page still showing">
  **Problem:** After adding DNS, domain still shows GoDaddy parking page

  **Solution:**
  1. Go to **My Products** > Find domain > Click **•••** (three dots)
  2. Select **Domain Settings** > **Manage**
  3. Scroll to **Forwarding** section
  4. If forwarding is active, click **Edit** > **Delete Forwarding**
  5. Wait 15 minutes for DNS to propagate
</Accordion>

    - Type: Select `TXT Record` from dropdown

**Common Issues:**

<Accordion title="Can't use @ for CNAME record">
  **Problem:** Namecheap doesn't allow CNAME on root (@)

  **Solution:**
  - You can't use CNAME for apex domain on Namecheap
  - Use `A Record` for `@` (root domain) instead
  - Use `CNAME Record` for `www` or subdomains only
  - If Vibey provides CNAME for root, ask support for A record IP address
</Accordion>

<Accordion title="Using custom nameservers">
  **Problem:** DNS records not saving or not working

  **Solution:**
  1. Check **Nameservers** section (above DNS records)
  2. If it says "Custom DNS" or shows external nameservers:
     - You're using a different DNS provider (like Cloudflare)
     - Add DNS records at that provider instead

     - Click **Nameservers** dropdown
     - Select "Namecheap BasicDNS"

</Accordion>

<Accordion title="Parking page redirect enabled">
  **Problem:** Domain redirects to Namecheap parking page

  **Solution:**
  1. On **Advanced DNS** tab, scroll to **Domain Redirect** section
  2. If redirect is active, click trash icon to delete it
  3. Ensure **URL Redirect Record** (if exists) is deleted
  4. Save changes and wait 15 minutes
</Accordion>

  <Step title="Add CNAME Record (for www subdomain)">
    Under **Subdomain Records (optional)** section:

    **Fill in:**
    - Subdomain: Type `www` (or your subdomain like `funnel`)
    - Record Type: Select `CNAME` from dropdown
    - IP Address or Target Host: Paste CNAME from Vibey (e.g., `cname.vibeos.com`)
    - Click **+ Add** button
  </Step>

  <Step title="Add TXT Record (for verification)">
    Under **Subdomain Records (optional)** section:

    **Fill in:**
    - Subdomain: Type `_vibey-verification` (or leave blank for root)

    - TXT Record: Paste verification code from Vibey (e.g., `vibey-verify-abc123xyz789`)
    - Click **+ Add** button
  </Step>

  <Step title="Save DNS Settings">
    Scroll to bottom of page

    Click **Save DNS** button

    Dynadot will show "DNS settings saved successfully" message
  </Step>

  <Step title="Verify Records">
    Your records should now appear in the DNS settings list

    Double-check:
    - A record for @ (root)
    - CNAME for www
    - TXT for verification
  </Step>
</Steps>

**Common Issues:**

<Accordion title="Domain forwarding interfering with DNS">
  **Problem:** Domain forwards to another site instead of using DNS records

  **Solution:**
  1. On domain management page, click **DNS** in sidebar
  2. At the top, check DNS setting dropdown
  3. If it says "Domain Forwarding" or "Stealth Forwarding":
     - Change to **Dynadot DNS**
     - Click **Save Settings**
     - Re-add your DNS records
  4. Forwarding overrides DNS records - must use Dynadot DNS
</Accordion>

<Accordion title="DNS changes not saving">
  **Problem:** After adding records, they disappear when you refresh

  **Solution:**
  1. Make sure you clicked **Save DNS** button at bottom of page
  2. Dynadot requires explicit save - records aren't auto-saved
  3. After saving, wait 1-2 minutes and refresh page
  4. Records should persist
  5. If still not saving, try different browser or clear cache
</Accordion>

<Accordion title="Subdomain vs Domain Record confusion">
  **Problem:** Not sure where to add A vs CNAME records

  **Solution:**
  - **Domain Record (optional)** section = Root domain (@)
    - Use for apex domain (example.com)
    - Add A record here
  - **Subdomain Records (optional)** section = All subdomains
    - Use for www, funnel, get, etc.
    - Add CNAME records here
    - Also add TXT verification here (unless Vibey says root)
</Accordion>

<Accordion title="Using external nameservers">
  **Problem:** DNS records can't be added in Dynadot

  **Solution:**
  1. Check **Name Servers** section on domain management page
  2. If it shows custom nameservers (not Dynadot's):
     - You're using external DNS (like Cloudflare)
     - Add DNS records at that provider instead
  3. To use Dynadot DNS:
     - Click **Name Servers** section
     - Select "Dynadot Name Servers"
     - Save and wait 15 minutes
</Accordion>

## Domain Verification

### Verification Process

1. You add DNS records
2. DNS propagates (5-60 minutes)
3. Vibey checks records every 5 minutes
4. Verification succeeds → SSL provisions
5. Domain status = "Verified"

### Domain Status

**Pending**
- Just added
- DNS not configured yet
- **Action:** Add DNS records

**Verifying**
- DNS records detected
- SSL provisioning in progress
- **Action:** Wait 5-15 minutes

**Verified**
- DNS correct
- SSL active
- HTTPS enabled
- **Action:** Assign to funnel and publish

**Error**
- DNS misconfigured or SSL failed
- **Action:** Check DNS records, click "Retry"

### Checking Verification Status

Settings > **Domains** > View status badge

**Status Colors:**
- Yellow = Pending
- Blue = Verifying
- Green = Verified
- Red = Error

## SSL Certificates

### Automatic HTTPS

Every verified domain gets free SSL certificate.

**What You Get:**
- Automatic certificate provisioning
- Auto-renewal (no expiration)
- HTTPS enforced (HTTP redirects to HTTPS)
- A+ SSL rating

**No Action Needed:** SSL is completely automatic.

### SSL Provisioning Time

**Normal:** 5-15 minutes after verification

**Slow:** Up to 60 minutes if DNS propagated slowly

**Failed:** Check domain status for error details

### Force HTTPS

All traffic automatically redirects from HTTP to HTTPS.

**Example:**
- User visits: `http://example.com`
- Auto-redirects to: `https://example.com`

No insecure connections allowed.

## Assigning Domains to Funnels

Once domain is verified:

<Steps>
  <Step title="Open Funnel">
    Funnels > Select funnel > Edit
  </Step>

  <Step title="Click Publish">
    Top right > **Publish**
  </Step>

  <Step title="Choose Custom Domain">
    Select **Custom Domain** (not Vibey subdomain)
  </Step>

  <Step title="Select Domain">
    Choose from dropdown of verified domains
  </Step>

  <Step title="Confirm">
    Click **Publish to Domain**
  </Step>
</Steps>

**Result:** Funnel live at `yourdomain.com` with HTTPS

### One Domain Per Funnel

Each funnel needs its own domain or subdomain.

**Example Setup:**
- Funnel 1 → `leadmagnet.example.com`
- Funnel 2 → `webinar.example.com`
- Funnel 3 → `book-call.example.com`

OR

- Funnel 1 → `example.com`
- Funnel 2 → `offer.example.com`
- Funnel 3 → `anotherdomain.com`

### Changing Domain

Switch funnel to different domain:

1. Funnel > **Publish Settings**
2. Select different domain
3. Save

Old domain shows 404. New domain goes live immediately.

## Troubleshooting

### Domain Stuck on "Verifying"

**Cause:** DNS not fully propagated or misconfigured

**Fix:**
1. Wait 60 minutes (DNS can be slow)
2. Check DNS with tool: whatsmydns.net
3. Verify records match exactly
4. Clear DNS cache: `ipconfig /flushdns` (Windows) or `sudo dscacheutil -flushcache` (Mac)
5. Click **Retry Verification**

### "DNS Records Not Found"

**Cause:** Records not added or incorrect

**Fix:**
1. Log into domain provider
2. Check DNS records exist
3. Verify exact match (name, value, type)
4. Save changes
5. Wait 10-20 minutes
6. Retry verification

### "SSL Certificate Failed"

**Cause:** CAA records blocking Vibey, or DNS proxy enabled

**Fix:**
1. Check for CAA records (DNS > CAA). If exists, add Vibey's CA.
2. If using Cloudflare, disable proxy (gray cloud, not orange)
3. Click **Retry SSL**

### Domain Shows "Not Secure" Warning

**Cause:** SSL not provisioned yet or certificate expired

**Fix:**
1. Check domain status in Vibey (should be "Verified")
2. Wait 15 minutes for SSL
3. Clear browser cache
4. If still broken after 1 hour, contact support

### Funnel Not Loading on Domain

**Cause:** Domain not assigned to funnel or DNS error

**Fix:**
1. Check funnel publish settings → correct domain selected?
2. Check domain status = "Verified"
3. Test domain directly: `yourdomain.com` (should load funnel)
4. Check browser console for errors

### "Domain Already in Use"

**Cause:** Domain assigned to another funnel in your account or another Vibey account

**Fix:**
1. Check all funnels → which one uses this domain?
2. Unassign domain from old funnel
3. Assign to new funnel

**If domain in another account:** You can't use it. Remove from other account first.

## Advanced Setup

### Multiple Domains to One Funnel

Want `example.com` AND `www.example.com` to go to same funnel?

**Setup:**
1. Add both domains to Vibey
2. Verify both
3. Assign primary domain to funnel
4. Use DNS redirect for secondary → primary

**Cloudflare Page Rule:**
- URL: `www.example.com/*`
- Setting: Forwarding URL (301)
- Destination: `https://example.com/$1`

### Subdomain for Each Funnel

Organize by subdomain:

**Examples:**
- `leadmagnet.example.com` → Lead magnet funnel
- `webinar.example.com` → Webinar funnel
- `book.example.com` → Call booking funnel

**Benefits:**
- Keep main domain free
- Easy to organize
- No additional domains to buy

### Using Root Domain + WWW

**Best Practice:** Pick one, redirect the other.

**Option 1:** Use root (`example.com`)
- Add A record for `@`
- Redirect `www` → root (at DNS level)

**Option 2:** Use www (`www.example.com`)
- Add CNAME for `www`
- Redirect root → www (at DNS level)

Most modern sites use root (no www). Cleaner URLs.

## Best Practices

### Use Descriptive Subdomains

Make purpose obvious from URL.

**Good:**
- `leadmagnet.example.com`
- `webinar.example.com`
- `free-guide.example.com`

**Bad:**
- `funnel1.example.com`
- `page.example.com`
- `f.example.com`

### Keep DNS Simple

Only add required records.

**Required:**
- A or CNAME (for domain)
- TXT (for verification)

**Optional:**
- MX (for email - keep existing)
- Other records (leave untouched)
